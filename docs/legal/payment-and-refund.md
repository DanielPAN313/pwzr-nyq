# 宁约球支付与退款说明草案

> 状态：上线前草案。当前项目仍为本地模拟支付，正式启用微信支付前需要补齐商户号、证书、验签、回调、退款回调和对账流程。

## 1. 支付范围

支付适用于场馆预订、球局报名、平台活动等需要提前占位的订单。

## 2. 当前技术状态

当前代码已经拆出：

- 预支付接口：`/api/sports-app/orders/:id/prepay`
- 支付确认接口：`/api/sports-app/orders/:id/pay/confirm`
- 微信支付回调预留：`/api/sports-app/payment/wechat/notify`

本地开发阶段返回模拟支付参数，不会真实扣款。

## 3. 正式微信支付前必须补齐

- 微信支付商户号。
- API v3 key。
- 商户私钥和证书序列号。
- 统一下单参数生成。
- `wx.requestPayment` 参数签名。
- 支付通知验签和订单幂等处理。
- 退款申请、退款通知、对账和异常订单处理。

## 4. 取消与退款规则草案

- 未支付订单：用户可直接取消，不产生退款。
- 已支付订单：在允许时间内取消后进入退款流程。
- 开场或预订前 24 小时内取消：可按规则扣减信用分。
- 开场或预订前 1 小时内取消：可按规则扣减更高信用分。
- 已核销订单：原则上不支持普通退款，特殊情况由客服或场馆处理。

## 5. 对账与争议

正式上线后应保留订单号、微信支付单号、退款单号、回调时间、订单状态变更记录和操作日志，用于财务对账、场馆结算和用户争议处理。
