# 宁约球小程序自测清单

这份清单用于微信开发者工具、本地后端和后续真机预览。每次合并功能或 UI 分支后，先跑命令检查，再按页面走一遍主流程。

## 1. 启动前检查

在项目根目录运行：

```bash
git status --short --branch
npm ci
npm run check
npm run dev
```

微信开发者工具导入：

```text
目录：pwzr-nyq/miniprogram
AppID：touristappid
后端服务：不使用云服务
```

开发工具里确认：

- 编译不报错。
- 控制台没有红色 JS 异常。
- 本地后端地址仍是 `http://localhost:4174`。
- 未注册小程序时允许使用模拟登录和模拟支付。

## 2. 底部 Tab

逐个点击底部：

- 首页
- 订场
- 球局
- 消息
- 我的

检查：

- 页面不会空白。
- tab 切换没有 JS 报错。
- 页面标题、按钮、列表内容没有明显重叠。
- 放大/缩小模拟器后，核心内容仍然一致可见。

## 3. 首页

检查：

- 统计卡片显示正常。
- 快捷入口能跳到对应页面。
- 待处理事项可以跳转。
- 推荐场馆能进入场馆详情。
- 推荐球局能进入球局详情。
- 后端关闭时能看到兜底内容和错误提示。

## 4. 订场流程

在订场页检查：

- 搜索场馆、区域或运动能过滤结果。
- 足球/篮球/全部筛选能切换。
- 无结果时能清空筛选。
- 加载失败时能重试。
- 点击场馆卡片能进场馆详情。

在场馆详情页检查：

- 场馆名称、区域、价格、联系方式、开放时段显示正常。
- 日期选择可用。
- 可预约时段可选择。
- 已占用时段不能提交。
- 生成待支付订单后跳到订单页。
- 无可约时段时有明确提示。
- 加载失败时能重试或返回订场页。

## 5. 球局流程

在球局页检查：

- 搜索球局或场馆能过滤结果。
- 运动筛选能切换。
- 无结果时能清空筛选或发起球局。
- 加载失败时能重试。
- 点击球局卡片能进详情。

在发起球局页检查：

- 发布预览会跟随输入实时变化。
- 校验清单能提示标题、场馆、人数、时间、费用。
- 人数少于 2 人不能提交。
- 开始时间早于当前时间不能提交。
- 场馆加载失败时能重试或去订场页。
- 发布成功后跳到球局详情。

在球局详情页检查：

- 场馆、时间、人数、费用显示正常。
- 报名进度条显示正常。
- 参与球友为空时有提示。
- 互评入口在开放时可点击。
- 加载失败时能重试或返回球局页。

## 6. 订单流程

在订单页检查：

- 空订单时能跳去报名或订场。
- 待支付订单能模拟支付。
- 已支付订单显示核销码。
- 可核销订单能模拟核销。
- 已核销订单能进入评价入口。
- 取消订单按钮可用。
- 从消息跳入订单时能定位关联订单。

## 7. 消息流程

在消息页检查：

- 未读/已读分组显示正常。
- 未读数量显示正常。
- 点击关联订单消息能跳到订单页。
- 点击关联球局消息能跳到球局详情。
- 空消息时能回首页或查看订单。
- 加载失败时能重试。

## 8. 我的页

检查：

- 用户资料、统计卡片和入口分组显示正常。
- 我的订单入口可打开订单页。
- 我的球局入口可打开我的球局页。
- 信用分入口可打开信用分页。
- 场馆端入口可打开场馆管理页。
- 合规说明入口可打开隐私政策、用户协议、支付说明和场馆合作摘要。

我的球局页检查：

- 全部、待处理、我发起、我报名、已完成筛选可用。
- 每场球局显示下一步动作。
- 点击卡片或动作按钮能进球局详情。

信用分页检查：

- 信用分、等级、进度条显示正常。
- 参与场次、到场核销、爽约次数显示正常。
- 信用规则和恢复建议显示正常。
- 信用记录能按类型分组。

合规说明页检查：

- 隐私政策、用户协议、支付说明、场馆合作四个 tab 可切换。
- 页面明确提示当前是开发版摘要。
- 页面明确提示上线前需要替换正式文本。

## 9. 场馆端

在场馆管理页检查：

- 三步引导显示正常。
- 入驻申请校验清单能提示缺失项。
- 入驻申请提交后能清空表单并刷新。
- 有自己场馆时能维护价格、联系方式、开放时段。
- 维护资料校验清单能提示缺失项。
- 核销码输入为空时不能提交。
- 近期订单为空时能跳去订场页或刷新。
- 加载失败时能重试。

## 10. 通过标准

本轮自测通过需要同时满足：

- `npm run check` 通过。
- 微信开发者工具编译不报错。
- 底部 5 个 tab 都能打开。
- 订场、球局、订单、消息、我的、场馆端主流程都能走通。
- 加载失败、空数据、无筛选结果、表单缺项都有明确提示。
- UI 同伴改动后没有删除 JS 依赖的事件名和数据字段。
