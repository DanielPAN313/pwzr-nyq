## 一、当前定位

宁约球手机端目前定位为“安卓端 MVP / 黑客松展示版”，目标是把原本的网页版能力适配成更像真实手机 App 的体验。

当前重点不是单纯把网页缩小，而是围绕真实用户的使用路径重新组织界面：

- 找附近球局
- 订场馆
- 发起组局
- 报名支付
- 到场核销
- 赛后互评
- 查看订单、信用分和个人资料

## 二、当前完成度

按黑客松展示标准，手机端当前大约为 80/100。

这个评分代表：如果用于黑客松现场演示，已经可以比较完整地展示产品概念、核心流程和手机端体验。

## 四、当前已发现并修复的问题

### 1. 手机端显示旧版本

曾经出现模拟器显示的不是最新版本的问题。

处理方式：

- 重新执行 Capacitor 同步
- 重新构建 APK
- 重新安装到模拟器

### 2. 球馆数据为空

曾经出现“以前有球馆，现在没有了”的问题。

原因：

- 手机端 APK 请求 `/api/sports-app/bootstrap` 时没有真正打到后端
- 后来又发现跨域请求头 `X-Username` 被 CORS 拦截

处理结果：

- 修复 API 地址
- 修复 Android 明文网络访问配置
- 修复后端 CORS 响应头
- 手机端订场页已恢复球馆列表

### 3. 订场筛选太挤

之前“推荐场馆”“区域”等文字有贴边和拥挤问题。

处理结果：

- 调整卡片内边距
- 调整筛选区布局
- 调整移动端文字和按钮间距

### 4. 手机端不像 App

之前整体感觉像“网页压缩到手机里”。

处理结果：

- 重构首页信息顺序
- 固定底部导航
- 强化手机端卡片布局
- 压缩球局卡片高度
- 调整“我的”页面为个人中心结构

## 五、当前仍然存在的不足

手机端目前适合演示，但还没有达到正式上架标准。

主要不足包括：

### 1. 后端仍然是本地开发模式

目前手机端依赖本地后端服务，正式版需要部署到线上服务器。

正式版应该使用类似：

`https://api.ningyueqiu.com`

而不是：

`http://10.0.2.2:4175`

### 2. 登录仍然是演示型

当前登录偏本地演示账号。

正式版建议接入：

- 手机号验证码登录
- 或微信登录

### 3. 支付还不是正式支付

目前“报名支付 / 订场支付”更偏流程演示。

正式版需要接入：

- 微信支付
- 支付回调
- 订单验签
- 支付成功后自动更新订单状态

### 4. 订单状态流还需要正式化

正式产品需要清晰的订单生命周期：

```text
created 创建订单
pending_payment 待支付
paid 已支付
confirmed 已确认
checked_in 已核销
completed 已完成
cancelled 已取消
refunded 已退款
```

### 5. 场馆和球局数据仍偏样板

正式运营需要：

- 真实场馆入驻
- 真实可订时段
- 场馆后台管理
- 球局审核
- 订单处理
- 退款规则

## 六、下一阶段建议

下一阶段不建议继续盲目加新功能，而是优先做“正式化三件套”。

### 第一阶段：线上后端

目标：手机 APK 离开本机电脑后也能正常使用。

要做：

- 云服务器部署 Node.js 后端
- 部署 MySQL
- 配置正式数据库
- 配置 HTTPS 域名
- 手机端 API 地址改为线上地址

### 第二阶段：真实登录

目标：用户可以用真实身份登录。

建议优先做：

- 手机号验证码登录

后续再考虑：

- 微信登录

### 第三阶段：真实订单支付和状态流

目标：让报名、订场、支付、核销、退款形成真正闭环。

要做：

- 订单状态机
- 支付接口
- 支付回调
- 核销流程
- 取消和退款规则

## 七、适合对外介绍的一句话

宁约球是一个面向南京高校和园区用户的运动约局与场馆预订 App，核心目标是把“找人、订场、报名、支付、核销、互评”整合到一条手机端闭环里，让临时运动局更容易组织，也让场馆资源更容易被使用。

## 八、当前结论

手机端已经从“网页压缩版”推进到了“可展示的安卓端 MVP”。

它现在适合用于：

- 黑客松展示
- 项目路演
- 给朋友或潜在用户看产品方向
- 继续做正式 App 的基础

但如果要正式上线，还需要优先完成：

- 线上后端
- 真实登录
- 真实订单支付和状态流
