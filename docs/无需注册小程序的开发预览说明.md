# 无需注册小程序的开发预览说明

现在先不注册微信小程序，也可以继续开发和演示。当前项目有两层：

- `miniprogram/`：真正的微信小程序工程，后面注册 AppID 后用微信开发者工具打开这里。
- `site/`：本地 H5 小程序预览，用浏览器模拟小程序手机端体验，适合现在先开发、改流程、改 UI、给同伴协作。

## 现在先用哪个

现阶段优先用 H5 小程序预览：

```bash
npm run dev
```

然后打开：

```text
http://localhost:4174/
```

它已经做了这些小程序化处理：

- 固定 390px 手机画布，放大、缩小、换电脑打开都尽量保持同一套手机布局。
- 底部 5 个主 Tab 模拟小程序 TabBar。
- 支持页面直达链接，刷新后不会丢当前页面。
- 浏览器返回/前进会同步应用页面，接近小程序页面栈体验。
- `npm run check` 会同时检查真实小程序目录和 H5 小程序预览约束。

## 常用直达链接

给同伴发这些链接，可以直接进入指定页面：

```text
http://localhost:4174/?page=home
http://localhost:4174/?page=venues
http://localhost:4174/?page=games
http://localhost:4174/?page=messages
http://localhost:4174/?page=me
```

扩展页面也可以直达：

```text
http://localhost:4174/?page=orders
http://localhost:4174/?page=credit
http://localhost:4174/?page=my-games
http://localhost:4174/?page=create
http://localhost:4174/?page=teams
http://localhost:4174/?page=demo
```

## 同伴改 UI 时怎么做

1. 先拉最新代码：

```bash
git pull origin main
```

2. 安装依赖：

```bash
npm ci
```

3. 启动本地预览：

```bash
npm run dev
```

4. 浏览器打开：

```text
http://localhost:4174/?page=home
```

5. 改完后运行：

```bash
npm run check
```

## 改哪些文件

如果只是改现在的 H5 小程序预览：

```text
site/sports-app.css
site/mobile-preview-lock.css
site/mobile-tabbar-motion.css
site/sports-app.js
```

如果是改真正微信小程序：

```text
miniprogram/pages/
miniprogram/app.json
miniprogram/app.wxss
miniprogram/utils/
```

现在还没注册小程序时，不影响先改 `site/` 里的体验。后面注册后，再把已经验证过的页面结构和交互迁移/对齐到 `miniprogram/`。

## 每次提交前必须确认

```bash
npm run check
```

看到这两行才算通过：

```text
Mini Program check passed.
H5 Mini Program preview check passed.
```
