# 无需注册小程序的开发预览说明

现在先不注册微信小程序，也可以继续开发和演示。当前项目有两层：

- `miniprogram/`：真正的微信小程序工程，后面注册 AppID 后用微信开发者工具打开这里。
- `site/`：本地 H5 小程序预览，用浏览器模拟小程序手机端体验，适合现在先开发、改流程、改 UI、给同伴协作。

## 现在先用哪个

现阶段优先用 H5 小程序预览：

```bash
npm run dev
```

然后打开：

```text
http://localhost:4174/
```

它已经做了这些小程序化处理：

- 固定 390px 手机画布，放大、缩小、换电脑打开都尽量保持同一套手机布局。
- 底部 5 个主 Tab 模拟小程序 TabBar。
- 支持页面直达链接，刷新后不会丢当前页面。
- 浏览器返回/前进会同步应用页面，接近小程序页面栈体验。
- 浏览器里提供 `wx` 兼容桥，常用 `wx.request`、`wx.showToast`、`wx.navigateTo`、`wx.switchTab`、`wx.getStorageSync`、`wx.requestPayment`、`wx.scanCode` 可以先跑通。
- 支持 `?path=pages/games/games` 这种接近真实小程序启动路径的方式，并提供 `wx.getLaunchOptionsSync()`、`wx.getEnterOptionsSync()`、`getCurrentPages()`。
- `npm run check` 会同时检查真实小程序目录、H5 小程序预览约束，并实际执行一次 H5 `wx` 桥接层 smoke test。

## 常用直达链接

给同伴发这些链接，可以直接进入指定页面：

```text

http://localhost:4174/?page=games
http://localhost:4174/?page=messages
```

```text
http://localhost:4174/?path=pages/messages/messages
```

## 同伴改 UI 时怎么做

1. 先拉最新代码：

```bash
git pull origin main
```

2. 安装依赖：

```bash
npm ci
```

3. 启动本地预览：

```bash
npm run dev
```

4. 浏览器打开：

```text
http://localhost:4174/?page=home
```

5. 改完后运行：

```bash
npm run check
```

## H5 里的小程序 API 兼容桥

`site/miniapp-bridge.js` 会在浏览器里创建 `window.wx`，用于模拟一部分微信小程序 API。它的作用是让开发方式更接近小程序：

```js
wx.showToast({ title: "保存成功" })

wx.switchTab({ url: "pages/games/games" })

wx.request({
  url: "/api/sports-app/bootstrap",
  success(res) {
    console.log(res.data)
  }
})

console.log(wx.getLaunchOptionsSync())
console.log(wx.getEnterOptionsSync())
console.log(getCurrentPages())
console.log(wx.getWindowInfo())
console.log(wx.getDeviceInfo())
console.log(wx.getAppBaseInfo())
console.log(wx.getMenuButtonBoundingClientRect())
console.log(wx.env.USER_DATA_PATH)

wx.login({
  success(res) {
    console.log(res.code)
  }
})

wx.getUserProfile({
  desc: "H5 预览登录",
  success(res) {
    console.log(res.userInfo)
  }
})

wx.requestPayment({
  timeStamp: String(Date.now()),
  nonceStr: "preview",
  package: "prepay_id=preview",
  paySign: "preview",
  success() {
    console.log("H5 预览支付成功，不会真实扣款")
  }
})

wx.scanCode({
  success(res) {
    console.log(res.result)
  }
})
```

注意：这个桥接层只用于本地预览。真正上线后，`miniprogram/` 里仍然会使用微信开发者工具提供的真实 `wx` 环境。

扫码模拟有三种方式：

```text
http://localhost:4174/pages/orders/orders?scan=NYQ-CHECKIN-DEMO
```

或者在浏览器控制台里先设置：

```js
wx.setStorageSync("mock_scan_result", "NYQ-CHECKIN-DEMO")
```

如果都没有设置，调用 `wx.scanCode()` 时会弹输入框让你填一个模拟扫码结果。

## H5 里的小程序生命周期模拟

`site/miniapp-bridge.js` 也提供轻量版小程序全局入口：

```js
App({
  globalData: { apiBaseUrl: "http://localhost:4174" },
  onLaunch(options) {
    console.log("preview launch", options)
  }
})

const page = Page({
  data: { count: 0 },
  onLoad(query) {
    console.log("preview page load", query)
  }
})

page.setData({ count: 1 })
console.log(getApp().globalData)
console.log(getCurrentPages())
```

这只是为了让开发写法更接近小程序。正式运行时，微信开发者工具会提供真实的 `App`、`Page`、`Component`、`getApp` 和生命周期。
